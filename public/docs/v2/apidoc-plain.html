Records / create

POST /students/:student_id/records
Create a new student

Full description¶ ↑

Used for creating a new user.

POST the new info along with the params to /records

Note¶ ↑

When a new Student info is given, id is empty. This empty id is posted

along with the rest of the parameters to the server. Server database fills the id

and the next time you query for this student record, it is returned with a proper id.

Examples

'user' : { 'host' : 'localhost:3000'}

Params

  • name : required [

    Must be a String

    ]

    Student name

  • grade : required [

    Must be a Integer

    ]

    Student grade (standard)

  • roll : required [

    Must be a Integer

    ]

    Student roll number

  • id : required [

    Must be a String

    ]

    Student record id (inside database)

  • email : required [

    Must match regular expression /^[a-zA-Z0-9]+@[a-zA-Z0-9]+\.[a-zA-Z]+$/.

    ]

    email should fit the regex

Records / update

PATCH /students/:student_id/records/:id
Update the student info

PUT /students/:student_id/records/:id
Update the student info

Full description¶ ↑

You can update a student using id.

E.g., if student id = “5a1bdd440e72df0a2a5783a4”

POST with updated data to htpp://www.example.com/id/5a1bdd440e72df0a2a5783a4

Every student record will have its own unique hash id.

Note: id comes from the database (MongoDB in this case) and varies for every record.

Errors

500 Internal server error 
:note: probably something wrong with your code

Records / destroy

DELETE /students/:student_id/records/:id
Delete a student

Full description¶ ↑

Send a DELETE request to /records/:id

E.g., if student id = “5a1bdd440e72df0a2a5783a4”

DELETE request will be sent to htpp://www.example.com/id/5a1bdd440e72df0a2a5783a4
